## 0. 프로젝트 한 줄

라즈베리파이5에서 **실시간 눈 깜빡임 검출**(MediaPipe 눈 랜드마크 → EAR → STD 4단계 상태머신)을 하되,
**개인별 EAR 캘리브레이션** + **눈 영역 품질에 따른 조건부 경량 SR(초해상화)** 을 우리 method 로 얹는다.
목표: **IEEE Sensors Letters 논문 1편 완성**. 비교 baseline = Carcellar et al., IEEE IICAIET 2024(Pi3B+, YOLOv5, 20초 배치, 85%).

**우리 신규성(컨트리뷰션):** "눈 픽셀 품질 기준 **조건부** 경량 SR(엣지 실시간) + 개인별 EAR 캘리브 + STD 판정"의 결합.
특히 매 프레임이 아니라 **눈이 작을 때만 SR을 켜는 조건부 게이트**가 효율(엣지 경량화)의 핵심.

## 1. 박사님이 제시한 방향 (최종)

- 공개 데이터셋으로 모델 학습·비교는 시간이 오래 걸림 → **이후 확장**. 우선 논문 1편 완성.
- 현재 파이프라인은 "기존 기술 조합"일 뿐 → **우리만의 method 필요**.
- 제안한 두 가지 method:
  1) **피험자별 EAR 캘리브레이션 단계 추가** — 개안 n초 EAR → 폐안 EAR → "깜빡이세요" 지시마다 STD가 1회로 세는지 검증, 피험자당 10회 반복, 모든 데이터 기록 → 통계 분석 → *임계값 근거*.
  2) **영상 품질(얼굴/눈 픽셀 기준)에 따른 조건부 SR** — 눈이 작거나(멀거나) 저해상도/저조도면 SR로 얼굴 영역 개선 후 EAR 계산. Pi에서 돌 경량 SR.
- **새 파이프라인:**
  0. 피험자별 EAR 캘리브(실험 준비) → 1. MJPEG 서버 스트림 → 2. 클라이언트 JPEG 수신 →
  3. 이미지 품질 평가 → (3.1 개선 필요: SR로 얼굴 개선 후 4로 / 3.2 불필요: 바로 4) →
  4. MediaPipe 입력 → 5. 눈 랜드마크 검출 → 6. EAR 계산 → 7. STD 업데이트 → 8. 깜빡임 판단
- **박사님 지정 즉시 할 일:** ① 관련 논문 2~3(유사성 높은 것) ② 관련 특허 2~3 ③ Pi5용 경량 SR 모델 찾기 ④ 실험 설계·지표 선정 ⑤ 실험 진행. + 위 파이프라인 그림.

## 2. 지금까지 한 것 (완료 ✅)

### 코드 (전부 컴파일·단위테스트 통과)
`src/eyeblink/` 패키지 + `src/scripts/`:
- `config` 상수/경로 (STD 임계값·스트리밍·캘리브·SR 실험 파라미터)
- `landmarks` MediaPipe FaceLandmarker + EAR + 2단계(개안/폐안) 캘리브
- `std` STD 4단계 상태머신(A개안→B감김→C폐안→D뜨기→A), 완전 순환 시 1회. 장시간 폐안(졸음) 실격.
- `metrics` 카운트 + BPM 롤링 집계 (+ 불완전 비율 옵션)
- `robust` 강건화 전처리 훅 (SR/저조도) — **아직 스텁**(여기에 SR 붙일 자리)
- `profiling` fps / 단계별 처리시간 계측
- `pipeline` EAR 스트림 → 캘리브 → EMA → STD → 카운트 글루
- `streaming/` MJPEG 서버(server.py) + 클라이언트(client.py)
- `experiment` 피험자별 캘리브 실험 기록·통계 코어
- `scripts/run_server.py` (맥 카메라→MJPEG), `run_live.py`(실시간+fps로깅), `calibrate.py`(피험자 캘리브 실험)
- (구방향 코드는 `src/_legacy/`)

### 엣지 실현성 실측 (bring-up 성공)
맥 웹캠 → MJPEG(Flask) → Pi5 클라이언트 → MediaPipe → EAR → STD.
- **FaceLandmarker: 평균 24.2ms/frame (≈41fps 상한), 파이프라인 실측 25fps.** → 실시간 여유 확인, SR 얹을 ~16ms headroom.
- 깜빡임 카운트 정상 동작. (상세: `실험로그.md`)

### 캘리브레이션 실험 도구 (박사님 method ① 구현 완료)
`scripts/calibrate.py` + `eyeblink/experiment.py`:
- 개안 5초 → 폐안 3초 → 개인 임계값 확정 → "지금 깜빡이세요" 10회 검증.
- 결과 CSV(results/): `calib_<subject>_frames.csv`(프레임별 EAR+phase), `calib_<subject>_trials.csv`(시행별), `calib_summary.csv`(피험자 요약: 임계값·EAR 평균±표준편차·정확도 누적).
- 단위테스트 통과.

### 조사·문서 (참고자료/)
- `유사연구_특허_정리.md` — 유사 논문 3 / 특허 3 + 차별점 (박사님 ①②)
- `SR모델_후보.md` — Pi5용 경량 SR (OpenCV dnn_superres FSRCNN/ESPCN 추천) (박사님 ③)
- `ROADMAP.md`, `선행연구_조사.md`, `관련연구_및_문제정의.md`, `실험로그.md`, `데이터셋_설명.md`, `안구건강_선행사례.md`, `Pi_실행_가이드.md`, `맥북_세팅_브리핑.md`

### 데이터셋 (박사님: 공개셋 학습은 이후로 → 이번엔 보조)
