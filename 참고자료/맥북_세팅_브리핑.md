# 맥북 세팅 브리핑 — 카메라(MJPEG) 서버용  ※ 맥북 Claude에게 그대로 전달

## 프로젝트 한 줄 요약
라즈베리파이5에서 **실시간 눈 깜빡임 검출(횟수/BPM) + 엣지 강건화(SR/저조도)** 연구(IEEE Sensors Letters 목표). 코드는 파이썬 패키지 `eyeblink/`(폴더 `src/` 안). 파이프라인: 카메라 → (MJPEG 서버) → 클라이언트 → MediaPipe FaceLandmarker → EAR → STD 4단계 상태머신 → 깜빡임 카운트. **카메라 위치와 처리 위치를 분리**하려고 MJPEG 서버-클라이언트 구조를 씀.

## 지금 상황
- 라즈베리파이5: OS 굽기·VNC/SSH 연결 완료(데스크탑에서). **Pi가 처리(클라이언트) 담당.**
- 데스크탑: Pi를 VNC로 보는 용도. 그대로 둠.
- 맥북: 이번에 **카메라 소스 = MJPEG 서버**로 쓰려 함. 아직 아무것도 설치 안 됨.

## 맥북의 역할 (중요)
맥은 자기 웹캠 영상을 네트워크로 흘려보내는 **서버만** 돌린다. **라즈베리파이 관련 소프트웨어(이미저·VNC 등)는 맥에 전혀 필요 없음.** Pi가 이 스트림을 받아 처리한다.
전제: 맥과 Pi가 **같은 WiFi/네트워크**에 있어야 함.

## 맥북에서 준비할 것 (Claude에게 도와달라고 할 목록)
1. Python 3.10+ 확인 (`python3 --version`).
2. 프로젝트 코드 받기: `src/` 폴더(특히 `eyeblink/`, `scripts/`, `requirements.txt`)를 맥으로 복사(git clone 또는 파일 전송).
3. 의존성 설치(서버만이면): `pip install flask opencv-python`
   - (선택) 맥 단독 테스트도 하려면 `pip install mediapipe` 추가.
4. macOS 카메라 권한: 시스템 설정 → 개인정보 보호 및 보안 → 카메라 → **터미널(또는 사용하는 IDE) 허용**.
5. 방화벽: 서버 첫 실행 때 "python이 들어오는 연결을 수락하도록 허용?" 팝업 → **허용**.
6. 맥 IP 확인: `ipconfig getifaddr en0`  (WiFi. 안 나오면 `en1`)
7. 서버 실행 (반드시 `src/` 폴더에서):
   ```
   python -m scripts.run_server
   ```
   → `http://<맥IP>:8000/stream.mjpg` 로 영상 스트리밍. 브라우저로 `http://<맥IP>:8000/` 열어 영상 뜨면 성공.

## Pi 쪽 (참고 — 맥에서 하는 것 아님)
```
python -m scripts.run_live --source url --url http://<맥IP>:8000/stream.mjpg --show
```

## 트러블슈팅 (Claude에게 물어볼 포인트)
- 브라우저에서 `http://<맥IP>:8000/`이 열리고 영상이 보이는가?
- 카메라 권한·방화벽 팝업을 허용했는가?
- 맥·Pi가 같은 WiFi인가? (다른 대역이면 접속 불가)
- 카메라가 여러 개면 `--camera 1` 등으로 지정.
- 맥 단독으로 파이프라인만 보고 싶으면(엣지 아님): `python -m scripts.run_live --source webcam --show`

## 자주 쓰는 명령 요약
- 서버(맥, 카메라): `python -m scripts.run_server`
- 실시간 처리(Pi): `python -m scripts.run_live --source url --url http://<맥IP>:8000/stream.mjpg --show`
- 맥 단독 테스트: `python -m scripts.run_live --source webcam --show`
