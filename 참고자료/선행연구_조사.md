# 선행연구 조사 — 라즈베리파이5 실시간 깜빡임 검출 + 엣지 강건화(SR/저조도)

갱신: 2026-07-16 · 새 방향(박사님 전사문) 기준

---

## 1. 배경과 조사 목적

목표 시스템: **라즈베리파이5에서 실시간·강건한 깜빡임 검출(횟수/BPM)** — MediaPipe FaceLandmarker → EAR → STD 4단계 판정, 개인 맞춤 캘리브. **핵심 기여는 저조도·저해상도로 랜드마크가 무너지는 환경에서도 검출되게 하는 경량 강건화(SR/저조도 보정)를 엣지에서 실시간 구동**하는 것.

조사 축: (A) EAR·실시간 깜빡임 검출, (B) 상태머신 4단계 판정, (C) 적응형 임계값·개인 캘리브, (D) 엣지·임베디드 구현, (E) 강건화 전처리(SR·저조도), (F) MediaPipe 엣지 성능, (G) 데이터셋.

## 2. (A) EAR·실시간 깜빡임 검출 — 핵심 방법 계보

| 연구 (연도) | 핵심 | 우리와의 관계 |
|---|---|---|
| ★ Soukupová & Čech, *Real-Time Eye Blink Detection Using Facial Landmarks* (2016) | EAR(Eye Aspect Ratio) 정의. 눈 뜨면 일정, 감으면 0으로 급락. SVM(스캐닝) + HMM(개인 적응) 검출. | 우리 EAR 파이프라인의 원천. 반드시 인용. |
| PyImageSearch dlib EAR blink (2017) | dlib 랜드마크 + EAR 임계값 연속프레임 규칙. 실용 표준 구현. | 구현 레퍼런스. |
| Fogelton & Benešová / Savitzky–Golay 필터 (Information, 2018) | EAR 시계열에 평활 필터를 적용해 노이즈에 강건한 검출. | EAR 후처리 강건화 참고. |
| ★ InstBlink / MPEblink (CVPR 2023) | one-stage 다인원 실시간 in-the-wild 깜빡임 검출(얼굴검출+추적+깜빡임 동시). MPEblink(686 영상, 8,748 blinks). | 최신 SOTA·대규모 데이터셋. 단 **GPU급 무거움 → 엣지 아님**(대조군). |

## 3. (B) 상태머신 4단계 판정 — STD의 근거

| 근거 | 내용 |
|---|---|
| FSM 기반 눈 상태 검출 (Sensors 2019, VR eyeblink detector 등) | 유한상태머신으로 open/close 판정. 깜빡임 = "Open→Closed" 후 "Closed→Open" 전이. 지속시간 = 두 전이 시간차. |
| 4단계(closing / pause=closed / opening) + 지속 100–400ms | 800–900ms 이상 폐안은 졸음·내려봄으로 구분. 프레임별이 아니라 상태 순서로 판정해야 오검출 방지. |
| 깜빡임 운동학 (Kwon et al., J. R. Soc. Interface 2013) | 600fps로 자발 깜빡임 측정: 닫힘 속도 > 열림 속도, 지속 ~0.5초. STD 4상태의 생리학적 근거. |

→ 우리 STD `A(개안)→B(감기)→C(폐안)→D(뜨기)→A`: 1초 폐안을 30프레임×"감김"=30회로 오카운트하는 문제, 실눈·노이즈 오검출을 방지. 전체 순환 시에만 1회 판정.

## 4. (C) 적응형 임계값·개인 캘리브

- 고정 EAR 임계값의 한계: 사람마다 눈 크기·평소 EAR이 다름(작은 눈·홑꺼풀은 뜨고도 낮게 나옴).
- Adaptive Eye Aspect Ratio 연구, Soukupová의 HMM(피험자별 자동 적응).
- **우리 방식(박사님):** 시작 시 몇 초간 개안/폐안 측정 → EAR **이동평균**으로 개인 기준점을 동적 설정. 고정 임계값 대신 개인화.

## 5. (D) 엣지·임베디드 구현 — 직접 baseline

| 연구 (연도) | 핵심·성능 | 한계 / 우리와의 관계 |
|---|---|---|
| ★ Carcellar et al. (IICAIET 2024) | YOLOv5, Pi 3B+ + 5MP, 20초 영상 배치, blink rate로 눈 피로, 정확도 85%. | 최근접 baseline. **횟수만·20초 배치·저정확(정상 재현율 ~44%)·강건화 없음.** |
| Reddy et al. (2019) | 깜빡임률 + 공막 면적, Pi, 83%. | 지표 단순, 배치. |
| Fatigue/Drowsiness on Pi/Jetson (2018–2024) | Pi3 97%, Pi4B CNN 90–94%. | 인접 도메인(졸음). 엣지 실현성 근거. |
| ★ Nighttime driver fatigue (Modelling, 2025) | **저조도 보정(Zero-DCE/RetinexNet) + joint 얼굴·랜드마크 + EAR** 3단계 실시간. | 우리 파이프라인 아이디어와 **가장 유사**. 단 운전·야간 응용, 엣지 Pi 아님 → 우리는 Pi5 온디바이스로 차별. |

## 6. (E) 강건화 전처리 — 우리 기여 후보 (★ 핵심)

### 해상도 축 — Super-Resolution
- ⚠️ "SR로 저해상도 얼굴 랜드마크 개선"은 **이미 포화**: Super-FAN(2018), EfficientSRFace(2023), low-res 랜드마크(2019).
- 경량/실시간 SR: **ESPCN(~21k params, sub-pixel), FSRCNN, edge-SR**(Pi CPU에서 2×/3× 부분 성공, 4×·고fps 어려움).
- ⚠️ 주의: 순차(SR→검출)가 결합(joint)보다 못한 사례, SR 영상이 학습분포와 달라 오히려 손해 → **SR이 실제로 검출을 돕는지 초기 검증 필수.**

### 조도 축 — 저조도 개선 (SR의 대안/보완)
- **Zero-DCE (TPAMI 2021):** zero-reference 경량 저조도 개선, 얼굴 검출 정확도 향상. 곡선추정 방식이라 가벼움 → 엣지 적합.
- 경량 실시간 저조도 네트워크(임베디드 자동차 비전, 2025).

→ 우리 기여 = "**엣지(Pi5) + 실시간 + 깜빡임 검출 응용 + fps·전력 실측**"의 조합. 해상도(SR)·조도(저조도) 중 무엇을(또는 둘 다) 쓸지는 실험으로 결정.

## 7. (F) MediaPipe 엣지 성능 (실현성)

- Face Mesh/Landmarker = MobileNetV2 + BlazeFace 기반 경량. ARM CPU(무 GPU)에서 **27–30fps**, 랜드마크 처리 121fps 사례. 468 랜드마크 중 눈 키포인트만 사용.
- → Pi5 실시간 구동 plausible하나 **실측 필요**(전처리 SR/저조도까지 얹으면 fps 하락 → 최대 리스크).

## 8. (G) 데이터셋 — IRB 없이 개발·검증

| 데이터셋 | 특징 | 용도 |
|---|---|---|
| Eyeblink8 | 4인, 408 blinks, 640×480, 머리움직임·표정 포함 | 검출기 개발·검증(현실적) |
| ZJU | 20인, 정면/위/안경/비안경, 30fps, 320×240 | 저해상도·안경 조건(그대로 저해상도) |
| RT-BENE (ICCVW 2019) | 실시간 깜빡임 추정 벤치·baseline | 성능 비교 |
| ★ mEBAL2 | 38인, RGB + **NIR** + EEG | 저조도(NIR) 시나리오 |
| MPEblink (CVPR 2023) | 686 영상, 8,748 blinks, in-the-wild | 대규모·야생 검증 |
| CEW / MRL Eye, TalkingFace | 눈 상태·단일 화자 | 보조 |
| NTHU-DDD / YawDD | 졸음(서서히 감김) 영상 | 프록시 |

활용: (1) 개발·검증을 IRB 없이, (2) 공개셋을 **다운스케일/암전**해 저해상도·저조도 테스트셋 생성 → **SR/저조도 유무 비교 실험**의 재료. 2인 실험은 최종 검증용.

## 9. 갭·기여 요약

- 엣지 깜빡임(Carcellar/Reddy): 횟수만·배치·저정확·**강건화 없음**.
- 강건 검출(InstBlink, 야간운전 파이프라인): **GPU급이거나 엣지 아님**.
- ★ **빈 땅:** 라즈베리파이5에서 STD 기반 실시간 깜빡임 검출 + 저조도·저해상도 강건화(경량 SR/저조도)를 **온디바이스·실시간**으로. 조합·엣지 실측이 신규성.
